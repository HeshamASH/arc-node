## 2026-04-16 - Consensus State Machine Boundary & BFT Divergence
**Vulnerability:** The interaction between `ArcBlockExecutor` (Rust/EVM) and `malachite-app` (CometBFT/Consensus) lacks strict atomic commit semantics for `SystemAccounting` state changes. If a block fails or is aborted late in the `finish()` lifecycle, the base fee and gas usage statistics may be permanently corrupted.
**Learning:** Consensus and Execution layers MUST share an explicit atomic boundary. Any EVM state modifications (like `store_gas_values`) performed *before* absolute consensus finality create a risk of state-root divergence. Because malachite requires 2/3rds agreement on the `app_hash` (state root), any perturbation from a dropped block or aborted transaction sequence will permanently fork the node.
**Prevention:** All system-level state changes must be staged in memory and strictly committed *only* when the block is irrevocably finalized by the consensus engine. Ensure `ArcBlockExecutor` treats `finish()` as a pure function returning a proposed state transition, not an immediate mutation of the underlying database.
