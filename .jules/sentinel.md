## 2024-05-24 - Extended BFT Proof Poisoning in Polka and Round Certificates
**Vulnerability:** The state-pollution bug involving `seen_validators.push()` occurring *before* signature verification (previously identified in `CommitCertificate`) also exists in sibling verification paths: `verify_polka_certificate` and `verify_round_certificate` in `malachitebft/crates/signing/src/ext.rs`.
**Learning:** Security anti-patterns involving state mutation prior to validation are often copy-pasted across sibling functions within the same module or trait implementation. When a critical protocol vulnerability is found in one verification routine, all parallel routines handling similar data structures must be aggressively audited for the exact same logic flaw.
**Prevention:** Enforce strict coding standards ensuring that state-tracking mechanisms (like "seen" arrays or hashmaps) are only updated *after* cryptographic validation succeeds. Utilize static analysis or custom linters to flag any mutable state updates that occur prior to a `.verify()` call within the same scope.
