## 2025-04-23 - BFT Proof Poisoning State Leak (Polka/Round)
**Vulnerability:** Similar to the previously discovered Commit certificate issue, the Malachite BFT consensus engine pushes a validator's address to the `seen_validators` array *before* cryptographically verifying the validator's signature in both `verify_polka_certificate` and `verify_round_certificate`.
**Learning:** This is a classic state leak inside an iterative processing loop. When early validation passes (duplicate check) and causes state mutation (`seen_validators.push()`), but a subsequent, more expensive check fails (signature verification), the mutated state is not rolled back. This allows attackers to "pre-poison" the duplicate check by submitting fake signatures for honest nodes.
**Prevention:** Iterative loops that process untrusted data (like network messages or certificates) must delay all state mutations until *after* all validation steps (including expensive cryptographic checks) have succeeded. Alternatively, temporary state arrays should only be updated on the success path.
