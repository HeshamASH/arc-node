## 2025-05-18 - [Investigation] PQ Precompile Attack Surfaces Secure
**Vulnerability:** None found in PQ Precompile (`crates/precompiles/src/pq.rs`).
**Learning:** The PQ precompile properly implements defense-in-depth principles: 1) Strict input length verification before underlying parser deserialization 2) Use of `try_from` replacing panicking code logic for `slh_dsa` structs, properly wrapped by `map_err` passing to EVM precompile execution handler 3) Total statelessness (no signature replay caching vectors on precompile level). The precompile also safely returns a native boolean, negating address spoofing or `NativeCoinControl` blocklist bypasses since it lacks underlying EVM address context logic natively.
**Prevention:** Maintain strict `try_from` usage instead of unwrapping on untrusted cryptographic payloads.
