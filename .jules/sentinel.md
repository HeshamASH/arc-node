## 2026-04-17 - Consensus State Corruption via Gas History Slot Aliasing
**Vulnerability:** The `SystemAccounting` precompile utilizes a Modulo 64 ring-buffer strategy for calculating the storage slot of historical gas statistics. Because it hashes `block_number % 64`, multiple distinct block heights alias to the exact same physical storage slot in the global state trie. This causes historical states to be overwritten by future blocks, corrupting the trie, breaking Merkle proofs for bridges, and guaranteeing sync deadlocks for nodes attempting to catch up.
**Learning:** Never use modulo-based ring buffers directly mapped to global state trie storage keys if the state must serve as a deterministic, historical source of truth for consensus validation or Merkle proofs.
**Prevention:** Remove the modulo operation (`block_number % 64`) and use the absolute `block_number` to ensure a unique, non-colliding storage slot for every block's gas statistics.
