## 2026-04-17 - 🛡️ Sentinel: BFT Proof Poisoning via State Leak on Error (NEW-17)
**Vulnerability:** A signature loop validation within `verify_commit_certificate` mutates local state (`seen_validators`) prior to mathematically verifying the cryptographic signature. When invalid signatures are deliberately swallowed (ignored), the local state remains permanently polluted.
**Learning:** Any validation loop that silently skips or ignores invalid elements creates a massive attack vector if it mutates state *before* validating the ignored elements. Attackers can leverage the "ignored" path to pollute the validator context and construct "Poisoned" inputs that cause the system to reject valid counterparts further down the pipeline.
**Prevention:** Strictly separate Side-Effects from Validation. A state variable tracking unique/valid elements MUST only be updated AFTER the element has passed all cryptographic and protocol validations.
